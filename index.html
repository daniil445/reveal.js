RegExp

RegExp is a text search and replace tool.

Different languages ​​implement the interface differently.

Let's look at the implementation in JS.

History

The origins of regular expressions lie in automata theory, formal language theory, and Chomsky's classification of formal grammars.

Mathematician Stephen Kleene later described these models using his system of mathematical notation called "regular sets."

Ken Thompson built them into the QED editor, and then into the ed editor on UNIX.

Use

Since then, regular expressions have become widely used in UNIX and UNIX-like utilities, such as expr, awk, Emacs, vi, lex, and Perl. Gradually spread beyond UNIX and now we can use them in our favorite JS!

In JS the tool allows to use 2 parameters:

  • Template
  • Flags

Entry point syntax

The injection point of regular expressions with strings is in the .match(RegExp) method

There is also .matchAll(RegExp)

Injection syntax allows 2 options

Dynamic

                    
                    let str = "Which tag do you want to find?";
                    re =RegExp("template", "flags");
                    str.match(re);

Static

                    
                    str.match(/template/flags);

Answer

If there are matches, it will return a list

If there are none, it will return null

Flags

Flags will affect the search and may contain additional. info

There are only seven of them in JavaScript

Main flags

  • i- search is case-insensitive
  • g- search looks for all subsequent matches
  • m-Multiline mode (^ $)
  • s-Enables "dotall" mode, . == \n
  • u-Enables full Unicode support
  • y-Search mode at a specific position in the text
  • d-With this flag, the result of the regular expression is placed in an array. This flag only provides additional information

Pattern

Parameter for entering matching conditions

Special characters

  • $& - inserts the entire match found
  • $` - inserts part of the string before the match
  • $' - inserts part of the string after the match
  • $n - inserts the contents of the n-th bracket group of the regular expression
  • $ - inserts the contents of the bracket group named name
  • $$ - inserts the "$" character

Class characters

Character class

  • \d ("d": from English "digital")
  • \s ("s": from English "space")
  • \w ("w": from English "word")

Exclusive character class

  • \D - all except number
  • \S - all except space
  • \W - all except word

\p- property

Allows to use a pre-known list of characters collected according to a certain criterion

Letters

  • L - all
  • Ll - lowercase
  • Lm - modifiers
  • Lt - uppercase
  • Lu - uppercase
  • Lo - other

Numbers

  • Nd - decimal digit
  • Nl - digits denoted by letters (Roman)
  • No - other

Standard abbreviations

  • Hex_Digit
  • for different languages
    • \p{sc=Cyrillic} for Cyrillic letters
    • \p{sc=Han} for Chinese characters

State checks

Checks are needed to determine states within the text

^ - beginning

 
                    let str1 = "Mary had a little lamb"; 
                    alert( /^Mary/.test(str1) ); // true

$ - end

 
                    let str1 = "it's fleece was white as snow"; 
                    alert( /snow$/.test(str1) ); // true

\b - word/number boundary

 
                    alert( "Hello, Java!".match(/\bJava\b/) ); // Java 
                    alert( "1 23 456 78".match(/\b\d\d\b/g) ); // 23.78

Enumeration []

[...] - "search for any character from the given ones" 


                    alert( "Top hop".match(/[th]hop/gi) ); // "Top", "hop"

[^...] - "search for any character except the given ones" 


                    alert( "alice15@gmail.com".match(/[^\d\sA-Z]/gi) ); // @ and .

Quantifiers {}

{n} - pattern multiplier


                    \d{5} - denotes exactly 5 digits, it is equivalent to \d\d\d\d\d
                    \d{3,5} - \d\d\d || \d\d\d\d || \d\d\d\d\d
                    \d{3,} - \d\d\d - infinity\d

Abbreviations


                    + <=> {1,}
                    alert( "+7(903)-123-45-67".match(/\d+/g) ); // 7,903,123,45,67

                    ? <=> {0,1}
                    alert( "Should I write color or colour?".match(/colou?r/g) );

                    * <=> {0,}
                    alert( "100 10 1 44".match(/\d0*/g) ); // 100, 10, 1
                    

                    \d <=> [0-9],
                    \w <=> [a-zA-Z0-9_]
                    \s <=> [\t\n\v\f\r ]

Escapes

'\' is used to escape characters [ ] \ ^ $ . | ? * + ( )


                    alert( "Chapter 5.1".match(/\d\.\d/) );

Or dinamical


                    let regStr = "\\d\\.\\d";
                    let regexp = new RegExp(regStr);
                    alert( "Chapter 5.1".match(regexp) );
